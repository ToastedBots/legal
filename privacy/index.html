ForwardLink Privacy Policy

Last updated: June 12, 2026

This Privacy Policy explains how ForwardLink, a Discord bot operated for the Forward Assault community, collects, uses, stores, and deletes data.

1. Data we collect and store

ForwardLink stores only the data needed to provide community support, moderation, boost reward, and staff activity features. Stored data may include:

  • Discord user IDs and usernames.
  • Staff role IDs and staff membership information.
  • Server boost log timestamps, boost actions, and boost counts.
  • Booster claim data, such as platform and submitted in-game name.
  • Pending rename request data and owner decisions.
  • Aggregate staff activity counts by user, date, and category.
  • Ticket metadata, such as ticket channel ID, creator ID, status, type, priority, timestamps, and close reason.
  • Ticket transcript markdown, which may include message content, attachment links, and embed counts from ticket channels.
  • Data deletion request records, including request scope, status, timestamps, and owner decision metadata.

2. Message content

ForwardLink uses Message Content access for ticket transcripts, limited staff activity classification, and staff-submitted boost import workflows. Ticket transcripts may store message content for support and moderation recordkeeping. Staff statistics generally store aggregate counts rather than full message text.

3. How we use data

Data is used to operate ForwardLink features, including support tickets, transcript access, boost tracking, booster rewards, staff reporting, moderation workflows, and owner-reviewed data deletion requests. Data is not sold, used for advertising, or used to train machine learning or AI models.

4. Data retention

Some data is stored longer than 30 days where needed for support, moderation, reward eligibility, historical records, auditability, and community operations. Ticket transcripts and boost or staff history may be retained until no longer needed or until an approved deletion request is processed.

5. Data deletion requests

Users can request deletion of stored activity data through ForwardLink's bot-integrated direct message workflow. In DMs, the privacy command provides this policy, the Terms of Service, and a button to submit a deletion request. The user completes a modal with the requested deletion scope and reason. The bot sends the request to the bot owner for review, and the user is notified by DM when the request is approved or denied. Approved requests are processed against the bot database.

6. Security

ForwardLink stores production data on encrypted storage at rest through host-level disk encryption. Access to stored data is limited to authorized operators who need it for support, moderation, maintenance, or compliance purposes.

7. Discord and GitHub

ForwardLink operates on Discord and uses Discord API data to provide bot features. This legal site is hosted through GitHub Pages. GitHub may process visitor information for security and service operation when this site is visited.

8. Changes

This policy may be updated when ForwardLink features, data handling, or compliance requirements change. The latest version will be available at this URL.

9. Contact

For privacy questions or deletion requests, use ForwardLink's privacy command in direct messages with the bot or contact the Forward Assault support staff through the official Discord support process.
